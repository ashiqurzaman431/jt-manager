JT Manager — One-Time Assignments Migration

Moves embedded assignments out of shared/taskManager/tasks and into your private shared/taskManager/assignments/{yourUid} subtree. Admin sign-in required. Safe to close and re-run — it re-checks state every time, it doesn't track "already ran."
⚠ This tool ATTRIBUTES every existing assignment record it finds to whichever admin account runs this migration — the old data model never recorded who created an assignment, so there's no way to recover true per-user ownership. If more than one person was assigning tasks before this change, review the preview table below before confirming; you may want to hand-edit afterward in the Firebase console.

Run this once. Running it again after a successful migration is harmless (there'll be nothing left to move), but running it on a database that's already mid-use by multiple users could re-attribute their assignments to you — check the preview count first.
